# CRM Sales Management System - Phase 1 (ระบบจัดการลูกค้าและโอกาสขาย)

ระบบจัดการลูกค้าสัมพันธ์ (CRM Sales Management System - Phase 1) พัฒนาด้วยเทคโนโลยีระดับสูงสำหรับองค์กร (Enterprise Grade) รองรับฟังก์ชันในการบริหารจัดการข้อมูลลูกค้าและโอกาสทางการขาย (Module 1 Customer + Module 2 Opportunity) พร้อมฟังก์ชันอำนวยความสะดวกครบครัน สวยงาม ใช้งานง่าย และรองรับระบบเชื่อมโยงข้อมูล Supabase Real-time พร้อมโหมดสำรองข้อมูลออฟไลน์อัจฉริยะ (Sandbox Fallback Mode)

## 🏢 สรุปฟังก์ชันการทำงานหลัก (App Key Features)

### 1. หน้ากระดานผู้บริหาร (Executive Dashboard)
* **KPI บัญชีการเงินสด**: แสดงผลรวมมูลค่าโอกาสทางการขาย (Total Opportunity Value), ยอดน้ำหนักโอกาสชนะจริง (Weighted Pipeline = มูลค่า × โอกาสสำเร็จ %), ยอดที่ปิดการขายสำเร็จ (Won), และยอดจำนวนลูกค้าบริษัทที่ยังร่วมธุรกิจ
* **แผนภูมิวิจัยเชิงลึก**: แสดงผลสรุปด้วยหน้าต่างลวดลายพาย (Pie Chart) และแถบริ้ว (Bar Chart) แยกรายละเอียดเป็นขั้นตอนการดำเนินงาน พร้อมกราฟแนวโน้มแสดงเป้าหมายวันปิดดีลสรุปสิ้นเดือน (Monthly Close Forecast)

### 2. โมดูลจัดการลูกค้า (Module 1: Customer Management)
* **ระบบรหัสอัจฉริยะ**: สร้าง รหัสลูกค้า (Customer Code) ให้อัตโนมัติในฟอร์แมต `CUS-000001` โดยอ้างอิงลำดับความยาวของแผงจริง
* **ฟอร์มจัดเก็บบันทึก**: รองรับการตรวจสอบข้อมูลอย่างละเอียด (Validation) ตั้งแต่ชื่อ, เลขประจำตัวผู้เสียภาษี 13 หลัก (Tax ID), กลุ่มประเภทอุตสาหกรรม, เครดิตเทอม และที่อยู่พัสดุ
* **ข้อมูลผู้ประสานงานในบริษัท (Contact Persons Tab)**: รองรับกฎข้อมูลแบบ One-to-Many โดยในบานหน้าต่างรายละเอียดของลูกค้า 1 ราย สามารถเพิ่มเติมชื่อพนักงานติดต่อ, ตำแหน่ง, เบอร์มือถือ และอีเมลเพื่อทำการจัดเก็บแก้ไขย้อนหลังได้ไม่จำกัดจำนวน

### 3. โมดูลจัดการส่วนลีดเสนอราคา (Module 2: Opportunity Management)
* **รหัสโครงการอัจฉริยะ**: รันรหัสให้อัตโนมัติในตระกูล `OPP-260001` (อิงตามรหัสปี ค.ศ. 2026 ปัจจุบัน)
* **กระดานดีลส่องความเสถียร (CRM Kanban Board)**: มาพร้อมกระดานบอร์ดคัมบังเต็มรูปแบบ รองรับการคลิกดีไซน์การย้ายสถานะแบบ Drag and Drop ระหว่างกลุ่ม Lead -> Qualified -> Proposal -> Negotiation -> Won ได้อย่างลื่นไหล พร้อมส่งสัญญายิงเก็บเข้าสู่ Supabase Cloud ทันที
* **กฎเกณฑ์ทางธุรกิจที่เข้มงวด (Strict Business Rule)**: ระบบกำหนดให้เปิดปุ่ม **"ออกใบเสนอราคา (Create Quotation)"** เฉพาะเมื่อโอกาสการขายได้รับเลือกให้อยู่ในสถานะ **Proposal (เสนอราคา)** เท่านั้น นอกเหนือจากนี้ปุ่มเชื่อมงานจะถูกจำกัดสิทธิ์เพื่อรักษาความปลอดภัยของกระบวนการทำข้อตกลง

### 4. รายงานผลและการส่งออกไฟล์ (Reports & Export Engine)
* **การกรองข้อมูลประเมินรายงาน**: สามารถคัดกรองข้อมูลตามช่วงเวลาก่อตั้ง, รายชื่อบริษัทคู่ค้า, สถานะ และพนักงานผู้ดูแลดีลได้อิสระ
* **Excel Export**: บรรจุตัวคำนวณถอดข้อมูลตารางทุกหน้าเป็นไฟล์ Excel ได้อย่างเสถียรด้วยชุดรหัส UTF-8 BOM ที่ทำให้เปิดภาษาไทยได้อย่างคมชัดไม่เพี้ยนใน Microsoft Excel
* **Print to PDF**: โครงสร้าง CSS และ Layout ได้รับการปรับแต่งให้ทนทานต่อรูปแบบการพิมพ์ (Print Media) ท่านสามารถกดสั่งพิมพ์ในหน้าต่างได้ทางปุ่มและดาวน์โหลดเพื่อส่งเป็นไฟล์เอกสาร PDF ประชุมบอร์ดบริหารในทันที

## ☁️ คู่มือการตั้งค่าเพื่อเชื่อมต่อกับ Supabase (Supabase Deployment Guide)

แอปพลิเคชันเวอร์ชันนี้มาพร้อมเลเยอร์ **CRM Hybrid Service** ซึ่งจะตรวจสอบตารางในโปรเจกต์ Supabase ส่วนตัวของคุณ หากยังไม่มีตาราง ระบบจะนำทางเข้าสู่ Sandbox Offline Mode ให้คุณได้ทดสอบฟีเจอร์อย่างปลอดภัย 100% ทันทีที่รันเสร็จ! และเมื่อคุณต้องการเชื่อมข้อมูลให้ไหลเข้าสู่ Supabase สดคลาวด์ของคุณให้ทำตามขั้นตอนด้านล่างนี้:

### ขั้นตอนที่ 1: การตระเตรียมโครงสร้างตารางข้อมูล (SQL Creation)
1. เข้าไปที่หน้าเว็บ **[Supabase Dashboard](https://supabase.com/)** แล้วเลือกโปรเจกต์ของท่าน
2. นำทางไปยังแถบเมนู **SQL Editor** จากหน้าต่างเมนูด้านซ้าย
3. คลิกสร้างคิวรีใหม่ โดยจิ้มที่ปุ่ม **New Query**
4. เปิดไฟล์และคัดลอกคำสั่งด้านในไฟล์ `/supabase_setup.sql` ในแอปพลิเคชันนี้ นำมาวางลงในพื้นที่เขียนโค้ด
5. กดปุ่ม **Run** ที่มุมล่างขวาเพื่อกวาดล้างและสร้างตารางสถิติ `customers` ชนิดคู่ความสัมพันธ์ตาราง `opportunities`
6. เสร็จสิ้น! ปัจจุบันระบบฐานข้อมูล Cloud ของท่านพร้อมรองรับข้อมูลงิ้วบริการอย่างเป็นทางการแล้ว

### ขั้นตอนที่ 2: ตั้งค่าและการทดสอบระบบซิงค์ (Database Switcher)
1. รันระบบพรีวิวแอปพลิเคชัน นำทางไปที่แท็บหน้าจอ **"ตั้งค่า Supabase / SQL"** ทางเมนูแถบข้าง
2. ระบบจะระบุสถานะสัญญาณให้เรียบร้อยว่าสามารถตรวจรับตารางในคลาวด์ได้สมบูรณ์หรือไม่
3. สลับปุ่มควบคุมแหล่งบันทึกเป็นแบบ **"Cloud Sync Mode"**
4. ทุกๆ กิจกรรมการพัดเปลี่ยน ลบ เขียน ฟอร์ม หรือการลากวางดีลใน Kanban จะถูกบันทึกส่งตรงขึ้นคลาวด์ของท่านอย่างทันท่วงที
